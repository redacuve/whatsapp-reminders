# Internacionalización (i18n)

## Supported languages

| Code | Language   | File             |
|------|-----------|------------------|
| `en` | English    | `src/i18n/en.ts` |
| `es` | Español   | `src/i18n/es.ts` |
| `pt` | Português | `src/i18n/pt.ts` |

Language is stored per-user in SQLite (`settings` table). It is resolved once in `main.ts` via `getLocale(language)` and attached to `MessageContext.locale`.

---

## Locale object structure

Each locale file exports a `Locale` object with four top-level sections:

### 1. `commandList`

Alias tables used **only** by the parser to resolve trigger words. Never used for display.

```ts
commandList: {
  reminder: {
    list: ['remind', 'reminder'],       // words that trigger the command
    subcommands: {
      list:   { list: ['list'] },
      delete: { list: ['delete', 'del'] },
      edit:   { list: ['edit'] },
      interval: { list: ['interval', 'every'] },
      weekly: { list: ['weekly'] },
      monthly: { list: ['monthly'] },
      weekdays: { list: ['weekdays', 'workdays'] },
      weekends: { list: ['weekends'] },
      help:   { list: ['help'] },
    },
  },
  // ...
}
```

### 2. `commands`

Display metadata shown to users in the help/remi views. Keys match `CommandKey`.

```ts
commands: {
  reminder: {
    name: 'Reminder',
    desc: 'Set and manage reminders',
    trigger: 'remind',
    emoji: '⏰',
    featureName: 'Reminders',
    featureDesc: "I'll ping you at the right time",
    subcommands: [ 'remind list — show pending', '...' ],
  },
}
```

### 3. `responses`

Every user-facing string and template function. Also contains NLP keyword arrays used by the parser fallback.

**Fixed strings:** `status`, `ping`, `unknown`, `langChanged`, `langInvalid`, `reminderEmpty`, `noteEmpty`, `journalEmpty`, …

**Template functions:** `pomodoroStarted(task, mins)`, `reminderAdded(text, when)`, `recurringAdded(text, desc)`, `noteAdded(text)`, `journalStats(stats)`, …

**NLP keyword arrays** (used by `parser/index.ts` fallback):

| Key | Purpose |
|-----|---------|
| `thanksKeywords` | Triggers thanks response |
| `howAreYouKeywords` | Triggers "how are you" response |
| `emojiKeywords` | Exact-match emoji reaction |
| `whoAreYouKeywords` | Triggers "who are you" response |
| `whatCanYouDoKeywords` | Triggers feature overview |
| `laughKeywords` | Triggers laugh response |
| `complimentKeywords` | Triggers compliment response |
| `sorryKeywords` | Triggers apology response |
| `greetingKeywords` | Used by `helloHandler` for greeting detection |
| `farewellKeywords` | Used by `byeHandler` for farewell detection |

### 4. Top-level locale fields

| Field | Type | Purpose |
|-------|------|---------|
| `code` | `string` | ISO code (`en` / `es` / `pt`) |
| `name` | `string` | Display name |
| `tomorrowKeywords` | `string[]` | Words meaning "tomorrow" for the time parser |
| `dayLabels` | `{ today, tomorrow }` | Formatted day labels for reminder display |
| `vagueHours` | `Record<string, number>` | Maps phrases like `"morning"` to hour `9` |
| `motivationalMessages` | `string[]` | Pool for `message` command |

## How to add a new language

1. Create `src/i18n/{code}.ts` implementing the full `Locale` type.
2. Export named const `export const {code}: Locale = { ... }`.
3. Add the code to `availableLocales` in `src/i18n/index.ts`.
4. Import and add the locale to the `locales` map in `getLocale()`.
5. Make sure all `commandList`, `commands`, and `responses` keys are present — TypeScript will report missing required fields.
